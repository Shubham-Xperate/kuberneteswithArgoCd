--------------------------------------------------------------------------
HOW TO ACCESS THE APP
--------------------------------------------------------------------------

Option 1) Quick local check via port-forward (no Ingress needed):

  kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "ecommerce-chart.fullname" . }}-web {{ .Values.web.service.port }}:{{ .Values.web.service.port }}

  Then open: http://localhost:{{ .Values.web.service.port }}

{{- if .Values.ingress.enabled }}

Option 2) Via the Ingress host (recommended -- exercises the full routing path):

  Prerequisite: the NGINX Ingress Controller must be installed in this
  cluster first (it is NOT part of this chart):
    helm upgrade --install ingress-nginx ingress-nginx \
      --repo https://kubernetes.github.io/ingress-nginx \
      --namespace ingress-nginx --create-namespace

  1. Add this host to your machine's hosts file, pointing at your local
     ingress controller's IP (127.0.0.1 for most kind setups, or the output
     of `minikube ip` for minikube):
       Windows: C:\Windows\System32\drivers\etc\hosts
       Linux/Mac: /etc/hosts

       127.0.0.1  {{ .Values.ingress.host }}

  2. Open: http://{{ .Values.ingress.host }}
{{- else }}

Ingress is currently DISABLED for this release (ingress.enabled=false in
your values). Use the port-forward option above, or re-install with
ingress.enabled=true.
{{- end }}
